# Project Rules for Claude / AI Agents

## Persian (RTL) Content Formatting — قانون اجباری

هنگام نوشتن متن فارسی (RTL)، هر توکن LTR — شامل نام تابع، شناسه، URL، مسیر فایل، کلمه انگلیسی، یا کد — باید:

- در **خط جداگانه** قرار گیرد
- با **خط خالی قبل و بعد** از بقیه متن فارسی جدا شود
- **هرگز** درون جمله فارسی به‌صورت inline قرار نگیرد

دلیل: ترکیب inline متن LTR و RTL باعث به‌هم‌ریختگی ترتیب خواندن (bidirectional algorithm) می‌شود و متن غیرقابل‌خواندن می‌گردد.

backtick دور کد این مشکل را حل **نمی‌کند** — فقط line break حل می‌کند.

### مثال نادرست    

اعتبار حساب با تابع `wto_get_apikey()` خوانده می‌شود.
 
### مثال درست

اعتبار حساب با تابع زیر خوانده می‌شود:

```
wto_get_apikey()
```

و سپس از option `wto_apikey` بارگذاری می‌شود.

### Scope

این قانون شامل تمام artifactهای فارسی پروژه است:

- `_bmad-output/project-context.md`
- اسناد BMad (PRD، architecture، stories)
- هر پاسخ chat که محتوای فارسی دارد
- کامنت‌های فارسی درون کد (اگر کنار متغیر/تابع انگلیسی هستند)
